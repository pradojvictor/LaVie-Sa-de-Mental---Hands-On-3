POST Criar Novo Atendimento

http://localhost:3000/atendimentos

Endpoint para criação de novo atendimento.

Conteúdo esperado no body:

  • paciente_id: int, requerido
  • data_atendimento: date, requerido (utilizar padrão norte americano)
  • obs: string, requerido

Modelo de conteúdo esperado no body para criação de novo atendimento:
{
"pacienteid": 1,
"dataatendimento": "2020-01-01T10:10:00z",
"obs": "Descrição do atendimento para o paciente com id: 1"
}

Ao ser feita a solicitação para Criação De novo Atendimento o servidor irá retornar um dos seguintes códigos de status:

  • 200: Sucesso da solicitação.
  • 400: Indica que a solicitação está incorreta.
  • 404: URL não encontrada.

Content-Type
application/json
Authorization
Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6NDQsIm5vbWUiOiJhZG1pbiIsImVtYWlsIjoiYWRtaW5AZW1haWwuY29tIiwiaWF0IjoxNjUzNTE3NTk5fQ.dov-Ut1EdyNsLewvLVmStkpG-vlaUHD1kWKuqdVgNFQ

{
  "paciente_id": "id_paciente",

PUT Atualizar Paciente

localhost:3000/pacientes/6

O endpoint Atualizar Pacientes é utilizado para alterar informações referentes ao cadastro de um determinado paciente.

É necessário inserir o ID do paciente na URL da requisição:

http://localhost:3000/pacientes/id_paciente

Modelo de URL para solicitação da atualização de um paciente:

http://localhost:3000/pacientes/1

O número '1' é a indicação de qual paciente deve ser atualizado.

Conteúdo esperado no body:

  • nome: string, requerido
  • email: string, requerido
  • idade: string, requerido (inserir ano de nascimento do paciente)

Modelo de conteúdo esperado no body para criação de novo atendimento:
{
"nome": "nomeatualizadopaciente",
"email": "exemplo@email.com",
"idade": 18
}

OBS.: As informações que não serão atualizadas não necessitam ser inseridas.

Ao ser feita a solicitação para Atualizar PAciente o servidor irá retornar um dos seguintes códigos de status:

  • 200: Sucesso da solicitação.
  • 400: Indica que a solicitação está incorreta.
  • 404: URL não encontrada.
